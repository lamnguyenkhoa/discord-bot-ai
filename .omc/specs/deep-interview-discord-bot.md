## Goal
Build a Python Discord bot that acts as a casual chat companion — responding to every @mention with a contextually-aware reply generated via OpenRouter LLM. The bot reads today's and yesterday's short-term memory logs before composing each reply, giving it a sense of conversational continuity across sessions.

## Constraints
- **Language:** Python
- **Discord library:** discord.py (bot token via environment variable)
- **LLM:** OpenRouter API, accessed via the `openai` Python SDK (OpenRouter-compatible base_url)
- **Model:** Configurable via `config.py` or `.env` — not hardcoded
- **Trigger:** Bot responds only when @mentioned in any channel
- **Memory — Short-term:** Daily log files (e.g., `memory/2026-04-01.md`) recording conversations
- **Memory — Auto-load:** Today's and yesterday's logs are automatically included in every API call
- **Memory — Summarization:** If a day's log exceeds a configurable threshold (default: 50 messages or ~4000 tokens), it is summarized before being included in context
- **Deployment:** Runs on user's local Windows laptop, 24/7; no Docker/cloud required
- **Bot persona / system prompt:** Configurable via a file (e.g., `system_prompt.txt` or `config.py`) — editable without touching code; defaults to a friendly, casual companion
- **API key storage:** OpenRouter key stored in `.env`, never hardcoded

## Non-Goals (v1)
- Long-term persistent MEMORY.md (phase two)
- Responding to messages that do NOT @mention the bot
- Slash commands or prefix commands (mention-only for now)
- Multi-server support requiring per-server memory isolation (single server assumed)
- Web dashboard or admin interface
- Voice channels

## Acceptance Criteria
- [ ] Bot connects to Discord and stays online (verified by Discord showing bot as "online")
- [ ] When any server member @mentions the bot with a message, the bot replies in the same channel within ~5 seconds
- [ ] The reply is generated by OpenRouter using the configured model (verifiable via OpenRouter usage dashboard)
- [ ] Today's and yesterday's `memory/*.md` log files are read and included as conversation context in each API call
- [ ] Each reply + the triggering message is appended to today's memory log file
- [ ] If today's memory log exceeds the summarization threshold, it is summarized (shorter file, same semantic content)
- [ ] The bot handles OpenRouter API errors gracefully (logs the error, sends a fallback message to Discord like "Sorry, I'm having trouble thinking right now")
- [ ] Model name is read from config, not hardcoded — changing it requires no code edits
- [ ] Bot does not crash or stop responding when left running overnight
